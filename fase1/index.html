    Capacitacion

    4.1 Etapa de capacitación Vídeo explicativo: https://youtu.be/7WXql8bwp6I

    Esta etapa tiene como propósito dotar, al personal encargado de aplicar la guía, de los conocimientos básicos necesarios sobre accesibilidad en sistemas web. Busca que comprenda cómo interactúan los usuarios con ceguera total con dichos sistemas, los fundamentos para garantizar la accesibilidad y cómo realizar estudios en esta materia.

    Para cumplir este objetivo, se desarrolló el "Curso introductorio a temas de accesibilidad web para personas con discapacidad visual". • Modalidad: Aprendizaje híbrido a través de una plataforma de e-learning. • Contenido: El curso consta de 11 temas que incluyen el uso de agentes de usuario (lectores de pantalla y navegación por teclado), directrices para crear sitios accesibles y metodologías de evaluación. El material didáctico se apoya en vídeos, infografías y prácticas.

    Los 11 temas que conforman el curso son: 1. El resultado esperado para los lectores de pantalla e identificación de problemas comunes de los lectores de pantalla (1 hrs). Incluye una tarea: Mediante la evaluación, con la herramienta WAVE proponer un cambio a un fragmento de código HTML que cause problemas de accesibilidad. Respuesta esperada: Se espera una respuesta apegada al acceso universal a sistemas en independencia de otros términos o definiciones apegadas a la dada por la WCAG.

    Proceso de Evaluación (Pasos a seguir) El procedimiento para esta etapa se estructura en tres pasos claros: 1. Diagnóstico: Se aplica un examen inicial de 8 reactivos (preguntas abiertas, opción múltiple y relación de conceptos) para medir el nivel de conocimiento previo de los desarrolladores. 2. Capacitación: Se imparte el curso descrito anteriormente. 3. Validación del Aprendizaje: Se aplica el mismo cuestionario inicial para comparar los resultados y verificar la adquisición de los saberes.
